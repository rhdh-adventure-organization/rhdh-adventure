Your team has a valuable **application skeleton**, scaffold, or archetype that aligns with a common architecture
in your organization. However, manually replicating this skeleton for every new application slows you down and
introduces inefficiencies. This challenge is about turning that manual process into a **streamlined, automated workflow**
using Red Hat Developer Hub **software templates**.

In this stage, you will:

* Create a **software template** based on your existing application skeleton, or
* Use a **software template** from the community, and
* Onboard new applications seamlessly by automating the creation of a **source code repository** and setting up
a **default dashboard** in the catalog, complete with documentation, dependencies (if applicable),
and API integrations (if applicable).

The stage is complete when:

* Your software template is successfully registered in the catalog.
* A new application or component is created using the template, onboarded into the catalog, and includes
all required resources (source code repository, documentation, dependencies, APIs).

With this challenge, you'll move beyond manual processes and embrace the power of automation.
Empower your team to onboard new applications quickly and consistently, reducing friction and ensuring every
project starts with the right foundation.
