🧞🕵🏻‍♂️ Where is my app? Solution

Welcome! Don’t worry about being here—it’s absolutely okay! This section provides an example solution to the challenge, offering a step-by-step implementation as a reference to guide you.

Remember, this is just one way to solve the challenge, and your approach might differ depending on your creativity or specific requirements. Feel free to explore and adapt the solution to make it your own. The goal is to help you understand the process and give you the confidence to tackle similar challenges in the future. Let' 's dive in together! 🧞🚀

🧞📝 Declare it Solution

Any component must be registered into the Red Hat Developer Hub catalog.

The Software Catalog as its name implies, is a catalog of software components. It is designed to replace ad-hoc, informal systems that do not scale to handle the sprawl of modern cloud-native application ecosystems. Maintaining a software catalog massively simplifies the Onboarding Process for new developers joining the organization, or a new project. The catalog lets you discover all registered components in the catalog, as well as explore the APIs, the teams that own and operate them, issues, CI/CD pipeline status, and technical documentation in a single easily accessible place. The component information is kept up to date by the respective teams (in their corresponding Git repositories), and Red Hat Developer Hub periodically pulls the latest metadata to maintain the information in near realtime.

Adding items to the catalog is done using YAML files, typically stored in source control systems like Git. These YAML files live alongside, and in the same source code repository as the source code of the software components. The way to declare a component to be loaded into the catalog is by the catalog-info.yaml file.

Here there is a minimum example describing a sample component, for example a library:

apiVersion: backstage.io/v1alpha1
kind: Component
metadata:
  name: sample-library
  description: |
    This is a sample library.
  links:
    - title: Red Hat Developer Hub Documentation
      url: https://docs.redhat.com/en/documentation/red_hat_developer_hub
  annotations:
    backstage.io/kubernetes-id: sample-library
    backstage.io/techdocs-ref: dir:.
    github.com/project-slug: rhdh-adventure-organization/sample-library
spec:
  type: library
  lifecycle: production
  owner: team-a

Choose a repository in your GitHub or GitLab instance and add a similar file in that repo.
